VGT Text-to-Gesture Product Plan

Deliver a usable product that converts VGT-oriented text into understandable hand/arm gesture playback in a web 2D avatar, with clear unknown-word fallback and measurable quality targets.

Delivery Phases

Phase 0: Product Definition and Acceptance Criteria
Phase 1: Linguistic and Gesture Data Foundation
Phase 2: Text-to-Sign Planning Engine (API)
Phase 3: Avatar Gesture Runtime (Web, 2D MVP)
Phase 4: Integration and End-to-End Flow
Phase 5: Quality, Evaluation, and Hardening
Phase 6: Deployment and Ops
Phase 7: Post-v1 Expansion Path

MVP Timeline

M1 Foundation: schema + seed lexicon + API contract locked.
M2 Prototype: text input produces playable avatar sequence.
M3 Quality Gate: benchmark meets comprehensibility + latency targets.
M4 Release: staging and production deployment with monitoring.

Risk Register

HIGH

Linguistic mismatch

Mitigation: phrase-level overrides and reviewer loop.

MEDIUM

Sparse lexicon

Mitigation: prioritize frequent concepts and visible fallback.

MEDIUM

Animation clarity

Mitigation: transition tuning from test feedback.

LOW

Scope creep

Mitigation: lock v1 to comprehension-first MVP.

Dependency Map

  • Phase 1 data foundation blocks Phases 2 and 5.
  • Phase 2 API output format blocks Phases 3 and 4.
  • Phase 3 player and Phase 2 API must converge before Phase 4 integration.
  • Phase 5 evaluation gates Phase 6 release.
